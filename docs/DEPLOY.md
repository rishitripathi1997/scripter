# AWS deployment (Phase 1)

Deploy ConnectX Scripts to a single EC2 instance with Terraform. Postgres runs in Docker on the same host; scripts and logs go to S3.

**Target:** personal / dev testing on AWS (`us-east-1`).

## What Terraform creates

| Resource | Purpose |
|----------|---------|
| EC2 `t3.small` | Runs Docker Compose (API + Postgres + nginx) |
| VPC + public subnet | Created automatically (your account has no default VPC) |
| Elastic IP | Stable URL for the app |
| S3 bucket | Script storage and run logs |
| IAM role | EC2 → S3 access + STS AssumeRole for user scripts |
| Security group | Port **80** open to `0.0.0.0/0`, SSH on port 22 |

Estimated cost: **~$18–20/month**.

## Prerequisites

1. **AWS CLI** configured for IAM user `scriper`:

   ```bash
   aws configure
   aws sts get-caller-identity
   # Account should show: 552879111942
   ```

2. **Terraform** >= 1.5:

   ```bash
   terraform -version
   ```

3. **This repo pushed to GitHub** — EC2 clones from `main` at first boot. Commit and push deployment files before `terraform apply`.

## First-time deploy

```bash
# 1. Push latest code (including terraform/ and docker-compose.prod.yml)
git add .
git commit -m "Add AWS Phase 1 Terraform deployment"
git push origin main

# 2. Configure Terraform
cd terraform
cp terraform.tfvars.example terraform.tfvars
# Optional: set ssh_public_key in terraform.tfvars to use your own key

# 3. Provision infrastructure
terraform init
terraform plan
terraform apply
```

Terraform prints outputs after apply. Save the sensitive admin password:

```bash
terraform output -raw admin_password
```

**Wait 5–10 minutes** on first boot while Docker builds images. Watch progress:

```bash
ssh -i terraform/keys/connectx-scripts-key.pem ec2-user@$(terraform output -raw public_ip)
sudo tail -f /var/log/connectx-user-data.log
```

When bootstrap finishes:

```bash
terraform output app_url
# Open in browser, log in with admin / <password from output>
```

## Redeploy after code changes

### Option A — GitHub Actions (Phase 6, recommended)

Push to `main` — the [Deploy to EC2](../.github/workflows/deploy.yml) workflow runs `deploy.sh` automatically.

Setup once: add `DEPLOY_HOST`, `DEPLOY_USER`, and `DEPLOY_SSH_KEY` secrets. See **[docs/CI.md](CI.md)**.

### Option B — Manual SSH

Push to GitHub, then SSH and run the deploy script:

```bash
ssh -i terraform/keys/connectx-scripts-key.pem ec2-user@$(terraform output -raw public_ip)
sudo /opt/connectx-scripts/scripts/deploy.sh
```

Or as a one-liner from your laptop:

```bash
ssh -i terraform/keys/connectx-scripts-key.pem ec2-user@$(cd terraform && terraform output -raw public_ip) \
  'sudo /opt/connectx-scripts/scripts/deploy.sh'
```

The script: `git pull` → `docker compose -f docker-compose.prod.yml up -d --build`.

## Useful commands

```bash
# App logs
ssh ... 'cd /opt/connectx-scripts && docker compose -f docker-compose.prod.yml logs -f api'

# Restart everything
ssh ... 'cd /opt/connectx-scripts && docker compose -f docker-compose.prod.yml restart'

# Check health
curl http://$(cd terraform && terraform output -raw public_ip)/health
```

## Teardown

```bash
cd terraform
terraform destroy
```

This removes EC2, EIP, S3 bucket (must be empty), IAM role, and security group.

## Security notes (personal dev)

- HTTP only (no TLS). Cookies use `SESSION_COOKIE_SECURE=false` so login works over plain HTTP.
- Port 80 is open to the internet. Fine for short personal testing; tighten before wider use.
- Secrets (`SECRET_KEY`, `APP_ENCRYPTION_KEY`, DB password) are generated by Terraform and written to `/opt/connectx-scripts/.env` on the instance.
- Change the admin password after first login (create a new admin user via API, or redeploy with a new password in user-data).

## Troubleshooting

| Problem | Fix |
|---------|-----|
| App not loading after 10 min | Check `sudo tail -f /var/log/connectx-user-data.log` on EC2 |
| `VPCIdNotSpecified` / no default VPC | Fixed in latest Terraform — `git pull` and re-run `terraform apply` |
| **502 Bad Gateway** on login | API container likely crashed — see below |
| `permission denied` on `.env` | Run with `sudo`, or `sudo chown -R ec2-user:ec2-user /opt/connectx-scripts` |
| Login fails / session lost | Confirm `SESSION_COOKIE_SECURE=false` in `/opt/connectx-scripts/.env` |
| `git pull` fails on deploy | Ensure repo is public, or add a deploy key to EC2 |
| S3 permission errors | Verify instance has IAM profile: `aws sts get-caller-identity` on EC2 |

### 502 Bad Gateway

The UI loads but `/api/*` returns 502 — nginx is up but the **API container is down**.

SSH in and inspect:

```bash
ssh -i keys/connectx-scripts-key.pem ec2-user@$(terraform output -raw public_ip)
cd /opt/connectx-scripts
sudo chown -R ec2-user:ec2-user /opt/connectx-scripts   # if .env permission denied
docker compose -f docker-compose.prod.yml ps
docker compose -f docker-compose.prod.yml logs api --tail 80
```

**Quick fix** (often a bad `.env` from special characters in secrets):

```bash
# Regenerate a safe encryption key (32 alphanumeric chars)
python3 -c "import secrets; print(secrets.token_urlsafe(32)[:32])"
# Edit .env — set APP_ENCRYPTION_KEY to that value, then:
docker compose -f docker-compose.prod.yml up -d --build api web
```

Or pull latest code and redeploy:

```bash
sudo /opt/connectx-scripts/scripts/deploy.sh
```
